1. Filosofia del Prodotto
MTG Collection Manager è un'applicazione personale per tracciare la collezione fisica di carte Magic: The Gathering e la loro distribuzione reale nei mazzi. Non è un deckbuilder: il suo scopo primario è sapere, in ogni momento, quante copie si possiedono di una carta, quante sono in uso e dove.
1.1 Principio fondamentale
REGOLA CORE
Una carta REAL assegnata a un mazzo consuma una copia fisica della collezione.
Una carta PROXY non consuma copie della collezione.
Non esistono altri stati: solo REAL e PROXY.
1.2 I due stati
Stato
Definizione
Effetto sulla collezione
REAL
Copia fisica realmente assegnata a un mazzo
PROXY
Carta usata nel mazzo senza consumare copie fisiche
1.3 Formula di disponibilità
FORMULA
copie_disponibili = copie_possedute − Σ(quantità REAL nei mazzi)
Il calcolo avviene tramite oracle_id, quindi ristampe diverse della stessa carta sono equivalenti ai fini della disponibilità.

2.3 Query disponibilità (SQL)
La query cardine del sistema — calcola la disponibilità per ogni carta nella collezione:
SQL
SELECT
  ci.card_id,
  ci.quantity_owned,
  COALESCE(SUM(dc.quantity) FILTER (WHERE dc.usage_type = 'real'), 0) AS qty_used,
  ci.quantity_owned
    - COALESCE(SUM(dc.quantity) FILTER (WHERE dc.usage_type = 'real'), 0) AS qty_available
FROM collection_items ci
LEFT JOIN cards c ON c.id = ci.card_id
LEFT JOIN cards c2 ON c2.oracle_id = c.oracle_id
LEFT JOIN deck_cards dc ON dc.card_id = c2.id AND dc.usage_type = 'real'
GROUP BY ci.id, ci.card_id, ci.quantity_owned;

5.2 Regole di non-blocco
L'app non deve mai bloccare l'utente. La filosofia è:
Avvisare sempre in modo chiaro
Suggerire l'azione correttiva più logica
Permettere di procedere anche in stato di warning
Il sistema PROXY è il safety net: qualsiasi carta può essere aggiunta a un mazzo

8. Algoritmo di Import Decklist
8.1 Formati supportati
Il parser deve gestire il formato testuale standard usato da Moxfield, Archidekt, EDHREC, etc.:
FORMATO INPUT
1 Sol Ring
4 Counterspell
1 Rhystic Study
// Commenti con doppio slash vengono ignorati
SB: 1 Brainstorm  ← righe con prefisso SB: (sideboard) opzionali
8.2 Pseudocodice algoritmo
ALGORITMO
Per ogni riga del testo:
  1. Skip righe vuote e commenti (// o SB:)
  2. Parsa «qty NomeCarta»
  3. Lookup oracle_id su Scryfall (nome → oracle_id)
  4. Se carta non trovata → aggiungi a lista errori, continua
  5. Calcola available = qty_owned − Σ(REAL esistenti per oracle_id)
  6. real_qty = min(qty, available)
  7. proxy_qty = qty − real_qty
  8. Inserisci in deck_cards:
     - Se real_qty > 0: una riga con usage_type='real', quantity=real_qty
     - Se proxy_qty > 0: una riga con usage_type='proxy', quantity=proxy_qty
8.3 Output anteprima import

Nome carta

Copie REAL assegnate (verde)
Copie PROXY assegnate (rosso)
Eventuali avvisi (carta non in collezione, carta non trovata)

10. Glossario
Termine
Definizione
REAL
Copia fisica della carta assegnata a un mazzo. Consuma una copia dalla collezione.
PROXY
Carta usata in un mazzo senza consumare copie fisiche (stampa, sostituto, token).
oracle_id
Identificatore univoco Scryfall per una carta, indipendente dalla stampa o edizione.
collection_item
Record che rappresenta quante copie fisiche si possiedono di una carta.
deck_card
Record che rappresenta la presenza di una carta in un mazzo con qty e usage_type.
overcommit
Stato in cui le copie REAL assegnate superano le copie fisicamente possedute.
disponibilità
qty_owned − Σ(REAL assegnate a tutti i mazzi). Può essere negativa (overcommit).
import
Processo di caricamento di una decklist testuale nel sistema.
salute mazzo
Percentuale di carte REAL sul totale. Un mazzo con 100% REAL è completamente fisico.
